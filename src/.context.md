This directory contains all reusable UI components organized by functional category. Each component is a templ-based template file paired with its compiled Go code. Components follow a settings-struct pattern for configuration.

## Summary

- control/ — Interactive control components (buttons, sliders)
- display/ — Display-only components (alerts, modals, sidebars, tags, toasts, accordions)
- form/ — Form input components (text inputs, selects, radio buttons, text areas)
- import/ — Export of shared dependencies (CDN links, library imports) as templ templates
- toolset/ — Utility functions for code minification using esbuild

## Constraints

- MUST use the settings-struct pattern: required fields at the top, optional fields (marked with // OptionalFields comment) below
- MUST pair each .templ file with an auto-generated _templ.go file (generated by the templ compiler)
- MUST NOT store logic in templ files beyond templating; use Go code for business logic

## Guidance

- SHOULD define constant values (sizes, variations, states) alongside the component's settings struct
- SHOULD use templ.Component for nested content to support composition
- SHOULD support Alpine.js bindings via TwoWayStatePath and OneWayStatePath fields when applicable
