Form input components for collecting user data. All components support Alpine.js two-way binding and have accessibility features (labels, hints, disabled states).

## Summary

- inputField.templ — Text input with configurable type (text, email, number, date, etc.), hints, and affixes
- inlineRadioGroup.templ — Horizontal radio button group with multiple options
- radioInput.templ — Single radio button input
- selectInput.templ — Dropdown select with option groups support
- textArea.templ — Multi-line text input with configurable rows

## Constraints

- MUST include InputName for form submission compatibility (FormData)
- MUST support InputId for optional DOM reference
- MUST support two-way binding via TwoWayStatePath for Alpine.js
- MUST include IsRequired flag and visual indicators

## Guidance

- SHOULD support hint/description text with configurable display (tooltip vs description)
- SHOULD support prefix and suffix affixes (left/right) with values or state binding
- SHOULD include proper error state styling and messaging support
